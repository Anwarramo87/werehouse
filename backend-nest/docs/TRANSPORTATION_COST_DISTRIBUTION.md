# توزيع تكلفة المواصلات بشكل ديناميكي

## المشكلة السابقة ❌
كان كل موظف يدفع نسبة ثابتة من التكلفة بغض النظر عن عدد الموظفين:
- موظف واحد: يدفع 16,000 ل.س
- موظفين: الأول يدفع 16,000 والثاني يدفع 8,000 (غير عادل!)

## الحل الجديد ✅
التكلفة تُقسم بالتساوي على جميع الموظفين تلقائياً:
- موظف واحد: يدفع 16,000 ل.س
- موظفين: كل واحد يدفع 8,000 ل.س
- 4 موظفين: كل واحد يدفع 4,000 ل.س

---

## مثال عملي

### البيانات الأساسية:
- **تكلفة الباص الكلية**: 20,000 ل.س
- **خصم الشركة**: 20%
- **التكلفة الصافية**: 16,000 ل.س (يدفعها الموظفين)

### السيناريوهات:

#### 1️⃣ إضافة الموظف الأول
```
عدد الموظفين = 1
التكلفة لكل موظف = 16,000 ÷ 1 = 16,000 ل.س

✅ يُضاف خصم "بدل مواصلات" للموظف الأول: 16,000 ل.س
```

#### 2️⃣ إضافة الموظف الثاني
```
عدد الموظفين = 2
التكلفة لكل موظف = 16,000 ÷ 2 = 8,000 ل.س

✅ يُحدّث خصم الموظف الأول من 16,000 → 8,000 ل.س
✅ يُضاف خصم جديد للموظف الثاني: 8,000 ل.س
```

#### 3️⃣ إضافة الموظف الثالث
```
عدد الموظفين = 3
التكلفة لكل موظف = 16,000 ÷ 3 = 5,333.33 ل.س

✅ يُحدّث خصم الموظف الأول من 8,000 → 5,333.33 ل.س
✅ يُحدّث خصم الموظف الثاني من 8,000 → 5,333.33 ل.س
✅ يُضاف خصم جديد للموظف الثالث: 5,333.33 ل.س
```

#### 4️⃣ إزالة موظف
```
عدد الموظفين = 2 (بعد الإزالة)
التكلفة لكل موظف = 16,000 ÷ 2 = 8,000 ل.س

✅ يُحذف خصم الموظف المُزال
✅ يُحدّث خصم الموظفين المتبقين: 8,000 ل.س لكل واحد
```

## الآلية التقنية

### عند إضافة موظف (`addPassenger`):

1. **حساب التكلفة الصافية**:
   ```typescript
   netCost = totalCost × (100 - companyDeductionPct) ÷ 100
   ```

2. **حساب التكلفة لكل موظف**:
   ```typescript
   costPerEmployee = netCost ÷ (عدد الموظفين بعد الإضافة)
   ```

3. **تحديث/إضافة الخصومات**:
   - لكل موظف موجود: تحديث الخصم الموجود
   - للموظف الجديد: إضافة خصم جديد

### عند إزالة موظف (`removePassenger`):

1. **حذف خصم الموظف المُزال**:
   ```typescript
   UPDATE EmployeeBonus 
   SET deletedAt = NOW() 
   WHERE employeeId = '...' AND bonusReason LIKE '%بدل مواصلات%'
   ```

2. **إعادة حساب للمتبقين**:
   ```typescript
   costPerEmployee = netCost ÷ (عدد الموظفين المتبقين)
   ```

3. **تحديث خصومات المتبقين**:
   ```typescript
   UPDATE EmployeeBonus 
   SET assistanceAmount = costPerEmployee
   WHERE employeeId IN (...)
   ```

## التعرف على الخصومات

الخصم يُعرّف بـ:
- **النوع**: `بدل مواصلات`
- **الملاحظات**: تحتوي على:
  - اسم الخط/المسار
  - رقم لوحة الباص

**مثال**:
```
bonusReason = "بدل مواصلات - خط صحنايا - الكسوة (123456)"
```

هذا يسمح بـ:
- ✅ تمييز خصومات المواصلات عن باقي الخصومات
- ✅ ربط كل خصم بباص محدد
- ✅ تحديث/حذف الخصومات بدقة

---

## الحالات الخاصة

### ✅ إعادة تفعيل موظف غير نشط
- يُعامل كإضافة موظف جديد
- يتم إعادة حساب التكلفة للجميع

### ✅ الموظف موجود مسبقاً
- رسالة خطأ: `Employee already on this bus`
- لا تحديث للخصومات

### ✅ الباص ممتلئ
- رسالة خطأ: `Bus is at full capacity`
- لا تحديث للخصومات

### ✅ فشل تحديث الخصومات
- العملية تستمر (إضافة/إزالة الموظف تتم بنجاح)
- الخطأ يُسجّل في console
- المدير يمكنه التعديل اليدوي لاحقاً

## الاختبار

### سيناريو الاختبار الكامل:

```bash
# 1. إضافة باص جديد
POST /api/transportation/buses
{
  "route": "خط صحنايا - الكسوة",
  "plateNumber": "123456",
  "totalCost": 20000,
  "companyDeductionPct": 20,
  "capacity": 10
}

# 2. إضافة موظف أول
POST /api/transportation/buses/{busId}/passengers
{ "employeeId": "EMP001" }
# ✅ خصم = 16,000 ل.س

# 3. إضافة موظف ثاني
POST /api/transportation/buses/{busId}/passengers
{ "employeeId": "EMP002" }
# ✅ خصم EMP001 = 8,000 ل.س
# ✅ خصم EMP002 = 8,000 ل.س

# 4. إضافة موظف ثالث
POST /api/transportation/buses/{busId}/passengers
{ "employeeId": "EMP003" }
# ✅ خصم EMP001 = 5,333.33 ل.س
# ✅ خصم EMP002 = 5,333.33 ل.س
# ✅ خصم EMP003 = 5,333.33 ل.س

# 5. التحقق من الخصومات
GET /api/discounts
# يجب أن تظهر 3 خصومات بقيمة 5,333.33 ل.س

# 6. إزالة موظف
DELETE /api/transportation/buses/{busId}/passengers/EMP003
# ✅ يُحذف خصم EMP003
# ✅ خصم EMP001 = 8,000 ل.س
# ✅ خصم EMP002 = 8,000 ل.س
```

# دليل ربط جهاز البصمة مع نظام الحضور (Biometric Integration)

هذا الدليل يشرح كيف تربط جهاز البصمة مع الباك-إند ليتم تسجيل الحضور والانصراف تلقائياً.

## 1) هل الباك جاهز؟

الباك **جاهز لاستقبال** سجلات الحضور عبر API، ويحتوي على:
- CRUD للأجهزة: `GET/POST/PUT /devices`
- تسجيل حضور يدوي/آلي: `POST /attendance`
- رفع دفعات حضور: `POST /attendance/upload`

لكن الباك **لا يحتوي** على ربط مباشر مع جهاز بصمة (SDK/Listener). الربط يتم عبر **خدمة وسيطة** (Middleware/Gateway) تقرأ من الجهاز وتُرسل للـ API.

## 2) الفكرة العامة للربط

الجهاز لا يتصل عادة بقاعدة البيانات مباشرة، بل:
1) **الجهاز يسجّل الحدث** (IN/OUT) ويُخرجه عبر SDK أو بروتوكول الشبكة.
2) **خدمة وسيطة** تعمل في الشبكة (Windows Service/Node/Python) تقرأ الأحداث.
3) الخدمة تُرسل الأحداث إلى الباك عبر `POST /attendance`.
4) الباك يحفظ الحدث ويحسب لاحقاً الغياب/التأخير ضمن محرك الرواتب.

## 3) الخطوات العملية (Step-by-step)

### الخطوة A: إضافة الجهاز في النظام
أضف الجهاز إلى قاعدة البيانات عبر:
```
POST /devices
```
Payload مثال:
```json
{
  "deviceId": "DEV001",
  "name": "Front Gate Scanner",
  "location": "Front Gate",
  "ip": "192.168.1.100",
  "port": 5000,
  "model": "ZK Teco"
}
```

### الخطوة B: إنشاء مستخدم خدمة (Service Account)
أنشئ مستخدم لديه صلاحية `edit_attendance` ليستطيع الإرسال إلى `POST /attendance`.
- الخيار الأفضل: حساب مستقل خاص بالجهاز.
- استخدم `POST /auth/login` للحصول على توكن.

### الخطوة C: تشغيل خدمة وسيطة (Gateway)
الخدمة تقرأ من الجهاز ثم ترسل للـ API.

**مخرجات الخدمة يجب أن تكون بهذا الشكل:**
```
POST /attendance
```
Payload مثال:
```json
{
  "employeeId": "EMP001",
  "timestamp": "2026-05-01T06:55:00.000Z",
  "type": "IN",
  "deviceId": "DEV001",
  "location": "Front Gate",
  "source": "device",
  "verified": true,
  "notes": "Biometric sync"
}
```

**المتطلبات:**
- `employeeId` يجب أن يطابق صيغة `EMP###`
- `type` يجب أن يكون `IN` أو `OUT`
- `timestamp` بصيغة ISO 8601

### الخطوة D: التحقق
تحقق من الإدخال عبر:
```
GET /attendance?employeeId=EMP001&date=2026-05-01
```

## 4) ملاحظات مهمة (Quality & Reliability)

- **Idempotency:** الجهاز قد يرسل الحدث مرتين. يُفضّل أن تقوم الخدمة الوسيطة بمنع التكرار عبر مفتاح:
  `deviceId + employeeId + timestamp + type`.
- **Timezone:** تأكد أن الخدمة الوسيطة ترسل وقت UTC أو تضبط التوقيت بشكل صحيح.
- **Network:** الباك يجب أن يكون قابل للوصول من نفس شبكة الجهاز.

## 5) ماذا عن endpoints البصمة الموجودة في Auth؟

الـ endpoints تحت `/auth/biometric/*` تخص **تسجيل دخول المستخدم عبر بصمة** داخل النظام، وليس استقبال بصمة من جهاز حضور. الربط مع جهاز الحضور يتم عبر `POST /attendance` كما في الأعلى.

## 6) خطة ربط مقترحة (Simple & Safe)

- شغّل Gateway على نفس الشبكة الداخلية.
- اجعل Gateway يسحب أحداث البصمة كل دقيقة.
- أرسل الأحداث إلى `/attendance`.
- سجّل failures محلياً لإعادة الإرسال في حال انقطاع الشبكة.

# 🔌 دليل ربط جهاز البصمة مع الموقع (Fingerprint Device Linking Guide)

هذا هو المجلد الأهم الذي يشرح المعمارية الشاملة حول كيفية ربط جهاز البصمة (Fingerprint Machine - ZKTeco أو ما شابه) مع الـ Website بشكل مثالي ومتزامن (Real-time).

## 🌍 المعمارية التقنية (Architecture)
نحن لا نتصل بجهاز البصمة من الـ Frontend (React/Next.js) مباشرة. بل نتصل به عبر خادم الـ Backend (NestJS).

**هناك طريقتان للربط، حسب إمكانيات الجهاز:**

### الطريقة الأولى: الاتصال المباشر عبر الشبكة المحلية TCP/IP (Pull Method)
(تستخدم إذا كان سيرفر الـ NestJS يعمل على نفس شبكة الجهاز أو لديه Static IP).
1. يتم تثبيت مكتبة مثل `node-zklib` في الـ Backend.
2. يتم تشغيل خدمة `Cron Job` كل 5 دقائق في NestJS.
3. يقوم الخادم بطلب الدخول للجهاز `zkInstance.connect()`.
4. يقوم بسحب سجلات الحضور الجديدة `zkInstance.getAttendances()`.
5. يخزنها في Database، ويقارنها مع الـ Shift لحساب التأخير.

### الطريقة الثانية والأفضل للويب: بروتوكول ADMS / Push SDK (Push Method) - 🌟 RECOMMENDED
(موصى بها لأنها Real-time ولا تستهلك موارد الخادم وتعلق أوتوماتيكياً بالموقع).
1. يتم ضبط إعدادات الـ `Cloud Server` في جهاز البصمة ليۆشر إلى رابط موقعك (مثال: `https://api.yourwebsite.com/api/biometric/push`).
2. عندما يضع الموظف إصبعه، يقوم الجهاز فوراً (Real-time) بعمل `HTTP POST Request` إلى الـ NestJS Endpoint الخاص بك.
3. الـ NestJS يستلم البيانات:
   ```json
   { "employeeId": "1005", "timestamp": "2026-05-31 08:05:00", "punchType": "CheckIn" }
   ```
4. الـ NestJS يقوم بتطبيق (المعادلات الموجودة في الملف الثاني) وحساب ما إذا كان متأخراً (Late).

## 🛠️ كيف يعمل النظام لحساب الراتب تلقائياً نهاية الشهر؟

### 1- تسجيل الموظف (Registration)
عند إضافة موظف جديد في نظام الـ Factory، تضع له الرقم الوظيفي `1005` (نفس الرقم في جهاز البصمة).

### 2- البصمة اليومية (Daily Routine)
كلما بصم الموظف عبر الـ Device المتصل بالـ Website. يُسجل الـ Backend وقت القدوم الفعلي، ويقارنه بوقت الوردية (Shift Schedule).

### 3- اتخاذ الأكشن على الراتب (Taking Action on Last Salary)
لقد طلبنا أن يكون النظام ذكياً ويأخذ أكشن تلقائي.
سيتم بناء خدمة في NestJS `PayrollService` تعمل يوم 1 من كل شهر تلقائياً:
- تقرأ راتب الموظف الثابت (مثلاً 5 مليون ل.س).
- تحصر كم دقيقة تأخير من جدول الحضور (مثلاً 120 دقيقة طوال الشهر).
- تحصر كم يوم غياب (مثلاً 1 يوم).
- تقوم بتطبيق **المعادلة الرياضية**:
  - `(5,000,000 / 30 / 8)` = أجرة الساعة `20,833 ل.س`
  - الخصم = (ساعتين تأخير * 20,833) + (1 يوم غياب * 2 يوم عقوبة * 166,666) -> = ~ `375,000 ل.س`.
- يقوم الـ Backend بإنشاء بيان الدفع (Pay Slip) النهائي بصافي `4,625,000 ل.س` ويحتفظ بحالة الخصم موضحة للتدقيق.
